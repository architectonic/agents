# Agents Doctrine

Agents are employees of the system, not sovereigns.

They may specialize local behavior, but they do not change company, organization, project, or constitution rules.

## Rules

1. Agent-local doctrine can narrow upstream doctrine, not weaken it.
2. Agent identity must state role, authority, escalation, privacy, and refusal posture.
3. Agent skill bundles must be pinned or vendored with provenance.
4. Agent knowledge sets must declare scope, access mode, freshness, and authority.
5. Agent model policies must respect trust tiers and data/action boundaries.
6. Agent autonomy is opt-in and must be justified by permissions, audit, rollback, and human review.
7. High-stakes agents must avoid claiming professional authority unless a real licensed/human authority grants and reviews that work.

## Factory vs instance

Public archetypes are reusable factories.

Local installed agents are specific workers inside an organization.

Runtime agents are executing sessions with concrete permissions and logs.
